2. BREACH RESPONSE PHASES

Response Timeline Overview:
• Phase 1: Immediate Response (0-24 hours)
• Phase 2: Investigation (24-72 hours)
• Phase 3: Notification (As required by law)
• Phase 4: Remediation (Ongoing)
• Phase 5: Post-Incident Review (Within 30 days)

PHASE 1: IMMEDIATE RESPONSE (0-24 HOURS)

Priority: Contain the breach and preserve evidence. Do NOT destroy any logs or data.
  • CONTAIN: Isolate affected systems immediately
  • PRESERVE: Do not destroy logs, emails, or any potential evidence
  • ACTIVATE: Alert Incident Response Team members
  • DOCUMENT: Begin incident log - record all actions with timestamps
  • ASSESS: Perform initial scope assessment
  • IDENTIFY: Determine what systems and data may be affected
  • SECURE: Change compromised credentials
  • NOTIFY: Alert executive leadership
  • ENGAGE: Contact legal counsel
  • INSURANCE: Notify cyber insurance carrier
  • VENDORS: Notify service providers if their systems are involved

PHASE 2: INVESTIGATION (24-72 HOURS)

  • Determine root cause of the breach
  • Identify all affected systems and databases
  • Determine if data was actually accessed, acquired, or exfiltrated
  • Assess whether encryption was in place and if it was compromised
  • Identify specific data elements involved
  • Determine number of individuals affected
  • Identify California residents affected
  • Determine if Sensitive Personal Information was involved
  • Engage forensics firm if needed
  • Document all investigation findings
  • Assess notification requirements
  • Prepare preliminary incident report

PHASE 3: NOTIFICATION

California Breach Notification Requirements (Cal. Civ. Code § 1798.82)

Notification is required when unencrypted personal information was, or is reasonably believed to have been, acquired by an unauthorized person.

Notification Triggers - Personal Information includes:
  • Name + SSN
  • Name + Driver's license/CA ID number
  • Name + Financial account number with access codes
  • Name + Medical/health insurance information
  • Name + Unique biometric data
  • Username/email + password/security question answers

Consumer Notification Content Requirements

Notification must be written in plain language and include:

  • Name and contact information of the reporting company
  • List of types of personal information involved
  • Date of breach (if known) or estimated date range
  • Date of notice
  • Whether notification was delayed due to law enforcement investigation
  • General description of the breach incident
  • Toll-free numbers and addresses of major credit bureaus (if SSN involved)
  • Toll-free number, email, and address for company contact
  • Advice on what consumers can do to protect themselves

Notification Methods

  • Written notice: Sent to last known mailing address
  • Electronic notice: If consistent with E-SIGN Act
  • Substitute notice: If cost exceeds $250,000, more than 500,000 consumers affected, or insufficient contact information

Attorney General Notification

REQUIRED: If more than 500 California residents are affected, you must submit a sample copy of the notification to the California Attorney General.

Submit to: https://oag.ca.gov/privacy/databreach/reporting

Notification Timeline

  • Notify in the "most expedient time possible"
  • Notify "without unreasonable delay"
  • May delay if law enforcement determines notification would impede investigation
  • Document any delay and reason

PHASE 4: REMEDIATION

  • Fix vulnerabilities that caused the breach
  • Update security measures and controls
  • Implement additional monitoring
  • Retrain employees if human error was involved
  • Review and update security policies
  • Review and update vendor agreements if vendor was involved
  • Consider offering credit monitoring/identity protection services
  • Set up dedicated response line/email for affected consumers
  • Monitor for misuse of compromised data

PHASE 5: POST-INCIDENT REVIEW (Within 30 Days)

  • Conduct post-incident review meeting
  • Document lessons learned
  • Update incident response plan based on lessons learned
  • Update security policies and procedures
  • Implement additional preventive controls
  • Schedule follow-up security assessment
  • Prepare final incident report
  • Archive all incident documentation
